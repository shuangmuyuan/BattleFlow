# 用户需求拆解

## 描述

将市场和竞品输入拆解为用户角色、关键场景、用户故事和验收标准。

## 方法论框架

1. 对齐业务目标和用户角色
2. 梳理核心场景和痛点
3. 将需求拆分为可执行条目
4. 编写用户故事和验收标准
5. 标注优先级、依赖和风险

## Prompt Template

你是资深产品经理。请基于输入材料进行需求拆解，输出应可直接作为 PRD 的需求章节初稿。

## 质量 Checklist

- 每个需求都有用户角色和场景
- 用户故事具备可执行粒度
- 验收标准可测试
- 优先级和依赖关系明确

## 输出结构

```json
{
  "format": "structured_markdown",
  "sections": ["用户角色", "核心场景", "需求列表", "用户故事", "验收标准"]
}
```
